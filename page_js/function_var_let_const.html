var

存在变量提升的情况

      
        <script>
          console.log(a) // undefined
          var a = 20

          //编译器会将其变成以下执行
          var a
          console.log(a)
          a = 20
        </script>

在函数中使用使用var声明变量时候，该变量是局部的

      
        <script>
          var a = 20
          function change(){
            var a = 30
          }
          change()
          console.log(a) // 20
        </script>

函数内不使用var，该变量是全局的

      
        <script>
          var a = 20
          function change(){
            a = 30
          }
          change()
          console.log(a) // 30
        </script>

函数本身也是一个值，也有自己的作用域。它的作用域与变量一样，就是其声明时所在的作用域，与其运行时所在的作用域无关

      
        <script>
          var a = 1;
          var x = function () {
              console.log(a);//1
          };
          function f() {
              var a = 2;
              x();
          }
          f() // 1
        </script>

let

let命令所在{}的代码块内有效

      
        <script>
        {
          let a = 20
        }
        console.log(a) // ReferenceError: a is not defined.
        </script>

不存在变量提升

      
        <script>
        console.log(a) // 报错ReferenceError
        let a = 2
        </script>

块级作用域内存在let命令，这个区域就不再受外部影响

      
        <script>
          var a = 123
          if (true) {
              a = 'abc' // ReferenceError
              let a;
          }
        </script>

let不允许在相同作用域中重复声明

      
        <script>
          let a = 20
          let a = 30
          // Uncaught SyntaxError: Identifier 'a' has already been declared
        </script>

const

const声明一个只读的常量，一旦声明，常量的值就不能改变

      
        <script>
          const a = 1
          a = 3
          // TypeError: Assignment to constant variable.
        </script>

之前用var或let声明过变量，再用const声明同样会报错

      
        <script>
          var a = 20
          let b = 20
          const a = 30
          const b = 30
          // 都会报错
        </script>

const实际上保证的并不是变量的值不得改动，而是变量指向的那个内存地址所保存的数据不得改动

      
        <script>
          const foo = {};

          // 为 foo 添加一个属性，可以成功
          foo.prop = 123;
          foo.prop // 123

          // 将 foo 指向另一个对象，就会报错
          foo = {}; // TypeError: "foo" is read-only
        </script>

var、let、const三者区别

  1. 变量提升
  2. 暂时性死区
  3. 块级作用域
  4. 重复声明
  5. 修改声明的变量
