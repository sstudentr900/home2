commit切換 checkout

commit切換後，若內容是你要的，就下git reset，回到那個版本吧。（記得先git checkout master回到原本的地方，再下git reset喔）

          
            git checkout
            <commit id>

檔案還原 reset

            
              //commit id -> 還原到指定Patch id
      git reset --hard
              <commit id>
                //還原全部檔案狀態, HEAD->目前所在的 patch
      git reset HEAD

      //還原指定檔案狀態
      git reset HEAD
                <file>
                  //還原上一個檔案狀態, HEAD^->上一個所在的 patch
      //HEAD^ -> 還原到上一個
      //HEAD^^ -> 還原到上上一個
      git reset HEAD^

      //--hard -> 強制清除修改的內容
      git reset --hard HEAD^

      //上面語法如果刪除錯了可以再用此語法還原
      git reset --hard ORIG_HEAD

git reset --hard 是不是先前的版本已經刪除回不去了呢？

查找HEAD的修改紀錄

                
                  git reflog

遠端 remote

                                
                                  //新增遠端 連結至github
      git remote add
                                  <short name>
                                    <repository URL>
                                      //显示所有远程仓库
      git remote -v

      //显示某个远程仓库的信息
      git remote show

      //查看分支
      git show-ref

      refs/heads/master => 推送到 master
      refs/heads/main => 推送到 main

      //修改remote(rename)
      git remote rename
                                      <short name>
                                        <new name>
                                          //刪除remote(rm)
      git remote rm
                                          <short name>
                                            //上傳分支
      //-u 等同於 --set-upstream，設定 upstream 可以使分支開始追蹤指定的遠端分支
      git push
                                            <reomte name>
                                              <branch name>
                                                //下載專案
      git clone
                                                <repo URL>
                                                  //同步遠端分支
      //git remote update 等同於 git fetch --all
      git remote update

      //同步遠端分支，並且移除過時的遠端分支
      //參數 -p 等同於 --prune
      git remote update -p

      //同步分支(將遠端分支拉下來與本地分支進行合併)
      git pull

      //強制更新遠端分支
      //參數 -f 等同於 --force，表示強制的意思
      git push -f
